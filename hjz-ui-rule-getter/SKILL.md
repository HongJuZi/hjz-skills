# HongJuZi UI Rule Getter Skill

此 Skill 用于从上传的UI设计图片分析出完整的UI设计规范，生成标准化的设计方案文档，供开发团队参考使用。

## 1. 核心分析流程

### 1.1 图片上传与分析
1. **图片输入**：用户上传UI设计稿图片（支持本地图片路径或网络图片URL）
2. **视觉分析**：分析图片中的色彩、字体、布局、组件等视觉元素
3. **规范提取与抽象**：从图片中提取设计规范。**重要提示**：在描述用途及场景时，必须将具体的业务名词（如“商品”、“商云购”、“推广人数”）抽象为通用的 UI 术语（如“主体标题”、“品牌标识”、“辅助数据”），或采用“抽象名称（示例业务名）”的格式。
4. **文档生成**：生成标准化的UI设计规范文档

### 1.2 分析维度

#### 1.2.1 色彩系统分析
- 主色调及色值提取
- 辅助色系列分析
- 中性色（背景色、文字色）识别
- 功能色（成功、警告、错误等）识别
- 色彩使用场景和比例分析

#### 1.2.2 字体规范分析
- 字体家族识别（中文字体、英文字体）
- 字号层级分析（标题、正文、说明、小字）
- 字重规范识别
- 行高和字间距分析

#### 1.2.3 图标规范分析
- 图标尺寸识别（不同场景的图标大小）
- 图标风格分析（线性/面性）
- 图标颜色规则
- 圆角和边框规范

#### 1.2.4 视觉效果分析
- 渐变效果参数（角度、色值）
- 阴影参数（偏移、模糊、扩散）
- 圆角系统
- 透明度和混合模式

#### 1.2.5 布局规范分析
- 页面边距
- 卡片间距
- 内容内边距
- 网格系统

#### 1.2.6 组件样式分析
- 按钮、卡片、列表等组件样式
- 交互状态（默认、悬停、点击）
- 响应式适配

## 3. 工具使用说明

### 3.1 图片上传
- **本地图片**：使用 `Read` 工具读取本地图片文件
- **网络图片**：使用 `WebFetch` 工具获取网络图片
- **网页截图**：使用 `Puppeteer` 工具截取网页截图

### 3.2 分析步骤
1. **图片输入**：用户提供图片路径或URL
2. **视觉分析**：分析图片中的设计元素
3. **规范提取**：提取色彩、字体、布局等规范
4. **文档生成**：生成标准化的设计规范文档
5. **代码示例**：提供SCSS变量和组件样式示例

### 3.3 输出结果
- 完整的UI设计规范文档（Markdown格式）
- SCSS变量定义
- 组件样式示例
- 设计建议和最佳实践

## 4. 应用场景

### 4.1 新项目启动
- 从设计稿快速生成UI规范
- 统一团队设计语言
- 为开发提供清晰的设计参考

### 4.2 现有项目重构
- 分析现有设计，提取规范
- 统一视觉风格
- 提高代码可维护性

### 4.3 设计评审
- 检查设计一致性
- 识别设计问题
- 提供改进建议

## 5. 最佳实践

1. **清晰的图片输入**：提供高分辨率、完整的设计稿
2. **多页面分析**：分析多个页面以获取完整的设计规范
3. **持续更新**：随着设计的迭代，及时更新规范文档
4. **团队协作**：设计和开发团队共同维护规范文档
5. **工具集成**：将规范文档集成到开发工作流中
6. **通用化描述**：生成文档时，应将具体业务（如“商云购”、“推广人数”）抽象为通用 UI 术语（如“品牌名称”、“辅助数据”），以确保规范的可复用性。

此 Skill 可以帮助团队快速从设计稿中提取规范，确保设计的一致性和开发的准确性，提高项目的整体质量和开发效率。
