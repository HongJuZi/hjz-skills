# HJZ SQL Executor

This skill specializes in database operations within the HongJuZi Framework ecosystem. It bridges the gap between POPO (Plain Old PHP Object) definitions and the actual Database Schema.

## Capabilities

1.  **Generate SQL**: Creates `CREATE TABLE` SQL statements from POPO files.
2.  **Execute SQL**: Runs `.sql` files against the configured database.

## Workflow

### 1. Generate SQL from POPO

When a POPO file is created or updated, use this skill to generate the corresponding SQL.

*   **Input**: Path to POPO file (e.g., `{backend_project_root}/config/popo/productpopo.php`)
*   **Logic**:
    *   Reads `$_table` to determine the table name (replacing `#_` with project prefix, usually `h_`).
    *   Reads `$_fields` to generate column definitions.
    *   **Mapping Rules**:
        *   `id` -> `int(11) unsigned AUTO_INCREMENT PRIMARY KEY`
        *   `uniacid` -> `int(11) unsigned NOT NULL DEFAULT '0'`
        *   `create_time` -> `datetime` or `TIMESTAMP`
        *   `status` / `is_*` -> `tinyint(1)`
        *   `_id` suffix -> `int(11)`
        *   `numeric` verify -> `decimal` or `int`
        *   Default -> `varchar(255)`
*   **Output**: Save to `{backend_project_root}/ai-coder/ai-ok/create_{module}_table_{timestamp}.sql`

### 2. Execute SQL File

Executes a specific SQL file against the project's database.

*   **Prerequisites**:
    *   Database configuration must exist in `{backend_project_root}/config/hconfig.php`.
    *   The SQL file must exist (e.g., generated by step 1).
*   **Process**:
    1.  **Read Config**: Parse `dbHost`, `dbUserName`, `dbUserPassword`, `dbPort`, `dbName` from `hconfig.php`.
    2.  **Confirm**: **MUST** ask the user for confirmation before execution, showing the target database and file path.
    3.  **Run Command**:
        ```bash
        mysql -h <dbHost> -u <dbUserName> -p<dbUserPassword> -P <dbPort> <dbName> < /path/to/your/file.sql
        ```
    4.  **Verify**: Check exit code to ensure success.

## Usage Examples

### Scenario A: New Module Setup

```bash
# 1. Generate SQL for 'Product' module
# (AI analyzes ProductPopo and writes SQL file)

# 2. Execute the generated SQL
# (AI reads config, asks for confirmation, then runs mysql command)
```

### Scenario B: Manual SQL Execution

If you have a hand-written SQL file (e.g., `update_schema.sql`):

1.  User: "Execute `update_schema.sql` on the dev database."
2.  AI: Reads config, confirms "Run `update_schema.sql` on `h_db` @ `127.0.0.1`?", then runs the command.
